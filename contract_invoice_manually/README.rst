This module extends ``Contract`` and adds the option "Manually Invoiced"
to a contract.

When invoices are generated automatically, "Manually Invoiced" contracts
are excluded.

They can be invoiced manually:

- Accounting >

  - Customers > *Manually Invoice Sale Contracts*
  - Vendors > *Manually Invoice Purchase Contracts*

The popup to invoice the contracts includes a filter, where the user can
further specify which contracts should be considered to invoice.

Use Cases / Context
===================

BUSINESS NEED:

- Some contracts should only be manually invoiced, for example if the
  data required to invoice is available only after the foreseen invoice
  date.

USEFUL INFORMATION:

- Extends the module ``Contract``
- Helpful modules: ``contract_sale``

Usage
=====

1. Go to *Invoicing* or *Accounting* > Settings > Contract > **Enable
   Manual Invoicing**
2. Go to *Sales* > Orders > Contracts > Contract > **Manually Invoiced**

|Contract Form|

3. Go to *Invoicing* or *Accounting* >

- Customers > **Manually Invoice Sale Contracts** or

- Vendors > **Manually Invoice Purchase Contracts** |Invoice Menu|
  *insert screenshot!*

- Select a date

- Check number of contracts to invoice |Wizard|

- Modify filter to further specify contracts to invoice |Wizard Filter|

- The *Manually Invoiced* filter can be removed if needed
